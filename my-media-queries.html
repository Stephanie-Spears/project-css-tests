Responsive Web Design

Responsive web design is when you code your site to change how it appears depending on the size of the screen it's being viewed on.

The screen being used to view the site is called the "viewport"

  • A media query is a block of CSS that is applied only when certain conditions about teh viewport are true. Ex: a media query that defines a max-width of 500px would only be applied when the user's viewport width is below 500px
  • Media Queriers are inside css style sheets, and since css cascades, they should be located at the bottom of the style sheet to prevent being overridden. They being with @media media-type and (max-width: 768px) {content}
  • the @media tells the browser it's a media query
  • the parenthesis defines when teh css in this query should be applied to our site
  • the curly braces contain the css that will be applied IF/WHEN the conditions we define are TRUE

Media queries often specify media type. it allows for the following options to be selected:

  • all: refers to any and all devices (default)
  • print: refers to viewing our site in "print preview" mode.
  • screen: refers to color screens.
  • speech: refers to screen readers which assist people with impaired site

Media Features

  • height: describes teh height of the viewport in pixels. This media feature can also have a min or max prefix added onto it. (ie. max-height which would define the maximum height a media-query's css should apply to)
  • width: describes the width of the viewport, in pixels. can also have a max-min prefix.
  • orientation: indicates whether the viewport is in landcape(wider than it is tall) or portrait mode(taller than it is wide)

Let's use the width media feature in our media query. We'll also include the MAX prefix to specify a maximum width.

@media screen and (max-width) {content}

  • the and between media type and the parentesis is for when we want to use both a media type and a media feature
  • the max prefix says that the CSS we eventually include in this media query should be applied to viewports ONLY within a max width of 768px

Breakpoints

a breakpoint is the point at which a media query's condition becomes TRUE. ex: our max-width of 768px will apply its styles ONLY WHEN the viewport is 768px OR LESS wide. 768px is therefore the BREAKPOINT because it's the point at which our query will be "ACTIVATED."

  • @media screen and (max-width: 768px) {body {background-color: black; color: white;}}
  • So if we have a width that is bigger than 768px, it will break back to default. If it is 768px or LESS, the background color and text color will change to what's inside the body bracket.
  • We can define multiple media queries for different size screens
  • When multiple media queries apply, the most recently applied media query's CSS will override the others IF they contain the same selectors and properties.
  • if we don't want to worry about MULTIPLE media queries applying at ONCE, then we can apply both a min and a max value to a single query.
  • @media screen and (max-width: 768px) and (min-width: 600px) { body { background-color: black; color: white;} }

WE CAN DEFINE ANY CSS IN MEDIA QUERIES

  • Oftentimes sites will display multiple columns on large screens, then condense them into a single column on smaller screens.
  • We can specify that any viewport ABOVE 768px should float our text into columns. This means any viewport BELOW the size of this breakpoint will display text in a single column
  • @media screen and (min-width: 768px) { .column {width: 300px; float: left; padding: 20px;} }
